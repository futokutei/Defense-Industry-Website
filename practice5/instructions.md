# Інструкція: Встановлення Node.js та NPM на macOS

Ця інструкція допоможе виправити помилку `zsh: command not found: npm`
шляхом встановлення середовища виконання Node.js.

------------------------------------------------------------------------

## Варіант 1: Через офіційний інсталятор

**Рекомендовано для новачків**

Найпростіший спосіб, схожий на встановлення звичайної програми.

1.  Перейдіть на офіційний сайт: **https://nodejs.org/**
2.  Натисніть велику зелену кнопку **LTS (Recommended for most users)**.
3.  Завантажиться файл з розширенням `.pkg`.
4.  Відкрийте його та дотримуйтесь інструкцій інсталятора\
    *(Continue → Install)*.
5.  Після завершення закрийте вікно інсталятора.

------------------------------------------------------------------------

## Варіант 2: Через Homebrew

**Для просунутих користувачів**

Якщо у вас встановлений пакетний менеджер Homebrew, це найшвидший
спосіб.

1.  Відкрийте термінал.

2.  Введіть команду:

    ``` zsh
    brew install node
    ```

3.  Дочекайтесь завершення встановлення.

------------------------------------------------------------------------

## Крок 3: Перевірка встановлення (ОБОВ'ЯЗКОВО)

Після встановлення (будь-яким способом) потрібно **перезапустити
термінал**, щоб система побачила нові команди.

1.  Закрийте термінал повністю: **Cmd + Q**.\
2.  Відкрийте його знову.
3.  Перевірте версії:

### Перевірка Node.js:

``` bash
node -v
```

Очікуваний результат: `v20.x.x` або подібне.

### Перевірка NPM:

``` bash
npm -v
```

Очікуваний результат: `10.x.x` або подібне.

------------------------------------------------------------------------

## Крок 4: Повернення до Практики 5

Тепер, коли `npm` працює, можна запускати проект.

### Перейдіть у директорію проекту:

``` bash
cd Defense-Industry-Website/practice5
```

### Встановіть залежності:

``` bash
npm install
```

### Запустіть сервер:

``` bash
node server.js
```

Якщо ви бачите повідомлення:

    Node Server running...

--- усе працює успішно!
