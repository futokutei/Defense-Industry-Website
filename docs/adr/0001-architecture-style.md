# 1. Вибір архітектурного стилю

* Статус: Прийнято
* Дата: 2025-11-28

## Контекст
Проект "Prod.Nexus" розвивається невеликою командою. Необхідно обрати архітектурний стиль для подальшого розвитку, балансуючи між складністю розробки та гнучкістю.

## Рішення
Ми обираємо **Модульний Моноліт**.

## Аргументація
1. **Простота:** Відсутність складної інфраструктури (Service Discovery, API Gateway, розподілені транзакції), яка необхідна для мікросервісів.
2. **Цілісність:** Легше проводити рефакторинг та змінювати межі модулів, поки вони в одній кодовій базі.
3. **Швидкість:** Швидший цикл розробки та деплою на поточному етапі.

## Наслідки
* Всі модулі працюють в одному процесі та використовують спільну БД (але логічно розділені).
* Комунікація між модулями відбувається через виклики методів (Service Layer), а не через мережу.

## Умови перегляду
Рішення може бути переглянуто, якщо:
* Кількість розробників перевищить 10 осіб.
* Окремі модулі вимагатимуть специфічного масштабування (CPU/RAM), відмінного від решти системи.
