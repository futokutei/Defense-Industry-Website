# Ubiquitous Language (Розширений Глосарій)

Цей документ описує єдину мову (Ubiquitous Language) системи **Prod.Nexus**.

## 1. Core Domain: Product Catalog (Каталог)
Це серце системи, те, заради чого вона існує — демонстрація продукції.

* **Product (Entity)**: Ключова сутність. Одиниця техніки (дрон, робот), що має унікальний `ID`. Життєвий цикл продукту керується адміністратором.
* **Product Description (Value Object)**: Текстовий опис характеристик продукту. Не має власного ідентифікатора, є частиною Продукту.
* **Product Image**: Файл зображення, що зберігається у файловій системі (`static/img`) та прив'язаний до продукту через шлях у БД.
* **Listing**: Відображення списку продуктів у вигляді карток на сторінці `/products`.

## 2. Supporting Subdomain: Marketing (Маркетинг)
Допоміжний домен для підвищення конверсії та залучення уваги.

* **Carousel (Aggregate)**: Колекція слайдів на головній сторінці.
* **Slide / CarouselItem (Entity)**: Окремий елемент каруселі. Має зображення, заголовок та кнопку дії (CTA).
* **CTA Button (Value Object)**: Елемент інтерфейсу на слайді ("Переглянути", "Детальніше"), що містить текст та посилання.
* **Text Position**: Налаштування візуального відображення (зліва/справа/центр).

## 3. Generic Subdomain: Identity & Access (Доступ)
Типовий функціонал для більшості систем, який ми реалізуємо через Flask-Login.

* **User (Entity)**: Користувач системи. У нашому контексті — це співробітник компанії.
* **Role (Value Object)**: Рівень прав доступу.
    * `Admin`: Може керувати контентом.
    * `Superadmin`: Може керувати іншими адміністраторами.
* **Session**: Тимчасовий стан авторизації користувача (зберігається в cookie).
* **Password Hash**: Зашифрований рядок пароля. Ми ніколи не зберігаємо чисті паролі.

## 4. Технічні терміни DDD (Архітектура)
* **Bounded Context (Обмежений контекст)**: Логічна межа, всередині якої модель має конкретне значення. У нас це `Admin Panel` та `Public Storefront`.
* **Repository**: Шаблон проектування, що забезпечує доступ до об'єктів домену (у нашому коді це методи `Query` через SQLAlchemy).
* **Upstream/Downstream**: Відносини між контекстами. `Admin Context` є Upstream (постачальником даних), а `Storefront` є Downstream (споживачем).
